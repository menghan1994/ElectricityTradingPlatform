# Product Scope & Phased Development

## MVP Strategy & Philosophy

**MVP 方式：问题解决型 MVP（Problem-Solving MVP）**

核心目标：用3个月时间交付一个可完整演示"AI比人工交易多赚钱"的端到端产品，让中石油试点公司看到真实数据验证结果后决定进入试点部署。

**团队配置：** 4人精英团队，经验丰富、实战能力强

**MVP 核心验证假设：**
1. AI策略（含储能联合优化）在日前报价场景下能比人工策略提升≥10%收益（回测验证）
2. 交易员愿意使用"多Agent协作建议+人工确认"模式完成日常报价和储能调度工作
3. 管理层认可回测报告的说服力并决定扩大试用
4. 基于规则的储能充放电策略能实现套利收益提升≥15%（vs 固定峰谷策略，回测验证）
5. 多Agent协作系统在电力交易决策场景中可稳定运行（协作成功率≥95%，96时段报价生成<30秒）

## Phase 1: MVP（3个月）

**支持的核心用户旅程：**
- ✅ 李娜成功路径：日前96时段AI报价建议+储能联合优化→审核微调→确认
- ✅ 李娜边缘场景：AI建议覆盖+理由记录+干预收益归因+储能调度调整
- ✅ 赵磊管理决策：多电站收益汇总+储能收益归因+回测报告导出
- ✅ 王总价值验证：历史回测演示+收益对比归因+储能资产价值量化
- ✅ 老周系统配置：引导式电站配置+储能参数配置+数据导入校验
- ✅ 李娜+陈工储能调度：日前储能充放电计划生成→确认→指令下发→执行

| 模块 | MVP必须交付 | MVP可简化 |

## Phase 2: Growth（6-9个月）

- 中长期合约智能优化（覆盖率推荐、分月分解优化）
- 储能高级优化：深度学习充放电策略（替代规则策略）、实时偏差跟踪与储能联动响应、储能EMS API实时对接
- 第三方功率预测模型接入（模型市场）
- 多省份市场规则适配
- AI可解释性升级：从规则描述→因子权重分析→完整Agent决策链路展示
- 信任度渐进 L2：常规时段AI自动填充待确认
- 多电站批量管理优化
- 人工干预收益归因深度分析
- 储能循环寿命经济性优化（充放电策略考虑电池衰减成本）
- 多Agent高级并行执行（多电站并行Agent图，不同电站的报价策略流水线式并行生成）
- MCP协议集成（Model Context Protocol标准化Agent工具接入）
- Agent性能优化（上下文压缩、智能缓存、Token消耗优化）

## Phase 3: Expansion（12-18个月+）

- 多市场联合优化调度（发-储-用-买-卖协同）
- 自动化交易提交（对接交易中心API）
- 辅助服务市场参与优化
- 移动端管理层看板
- 信任度渐进 L3：低风险时段Agent自动执行，仅异常时段人工干预
- 行业SaaS平台，服务多家大型能源集团
- 免费回测获客工具标准化
- 强化学习融合Agent（将学术界MARL方法融入LangGraph架构，从"LLM辅助决策"演进为"AI自主优化"）
- 跨市场多Agent联动（电力现货、辅助服务、绿电交易等多市场Agent协同策略优化）
- 电力交易专属小语言模型（SLM）训练，降低对通用大模型的依赖

## 明确不在MVP范围内（Out-of-Scope）

以下功能明确不在MVP范围内，避免范围蔓延：

| 排除项 | 归属阶段 | 说明 |
|--------|---------|------|
| 中长期合约智能优化 | Phase 2 | 覆盖率推荐、分月分解优化 |
| 实时偏差跟踪与储能联动响应 | Phase 2 | 需要实时数据通道，MVP仅支持日前调度 |
| 深度学习储能充放电策略 | Phase 2 | MVP采用规则策略保底 |
| 储能EMS API实时对接 | Post-MVP | MVP采用文件导入方式 |
| 第三方功率预测模型接入 | Phase 2 | MVP仅1个自有模型，接口预留 |
| 多省份市场规则同时适配 | Phase 2 | MVP仅1省 |
| 碳减排量记录与ESG报告 | Phase 2 | 领域需求已定义，MVP不实现功能 |
| 绿证管理与交易 | Phase 3 | MVP仅数据模型预留字段 |
| 自动化交易提交（对接交易中心API） | Phase 3 | MVP为人工提交 |
| 移动端看板 | Phase 3 | MVP为响应式Web |
| 辅助服务市场参与 | Phase 3 | MVP仅日前现货市场 |
| Agent自学习/强化学习优化 | Phase 3 | MVP为固定策略Agent |

## Risk Mitigation Strategy

**技术风险：**

| 风险 | 概率 | 缓解措施 |
|------|------|---------|
| AI报价引擎算法精度不足 | 中 | 先用规则+简单ML模型保底，迭代优化算法；回测环境充当持续验证工具 |
| 功率预测模型初期准确率不达预期 | 中 | 接口标准化设计，可快速切换或融合多模型；与客户原有预测对比验证 |
| 4人团队3个月交付压力（含储能+多Agent模块） | 中高 | 团队经验丰富；储能MVP采用规则策略+文件导入简化范围；多Agent采用12周分阶段路线图（第1-2周基础验证→第3-6周核心功能→第7-12周产品化）；可解释性等非核心UX先简化 |
| 储能规则策略效果不及预期 | 中 | 回测环境验证规则策略基线效果，Phase 2升级ML模型进一步优化 |
| 多Agent协调失败导致报价质量差 | 中 | Supervisor不一致检测+自动重新调用Agent；极端情况回退到规则引擎 |
| LLM输出不稳定（非法96时段报价） | 中高 | JSON Schema结构化输出验证+Guardrails+超限自动拦截 |
| LLM Token成本失控 | 中 | Langfuse全链路监控+单次调用上限+缓存复用（可降低50%成本） |
| LangGraph学习曲线延迟进度 | 中高 | 前2周专注培训+从2-3个核心Agent原型开始+官方教程和示例项目 |

**市场风险：**

| 风险 | 概率 | 缓解措施 |
|------|------|---------|
| 回测结果不及预期（<10%收益提升） | 低 | 多维度回测（日前报价+偏差优化），寻找收益最大的切入点 |
| 交易员抵触AI | 中高 | L1全人工审核模式+可解释性+"AI vs 人工"每日对比看板 |
| 中石油试点决策周期长 | 高 | 回测报告作为加速器，用数据缩短决策链路 |

**资源风险：**

| 风险 | 概率 | 缓解措施 |
|------|------|---------|
| 团队成员变动 | 低 | 核心模块文档化，接口标准化降低耦合 |
| 数据获取延迟 | 中 | 用公开数据+模拟数据先行开发，真实数据到位后替换验证 |
