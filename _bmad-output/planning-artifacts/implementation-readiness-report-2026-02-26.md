## PRD 分析

### 功能性需求（Functional Requirements）

**功率预测管理：**
- FR1: 交易员可查看指定电站的96时段功率预测曲线及置信区间
- FR2: 交易员可切换查看不同预测模型的预测结果（MVP仅1个自有模型）
- FR3: 系统管理员可配置功率预测模型的接入参数并监控运行状态，异常时系统自动告警
- FR4: 系统通过标准化API接入自有功率预测模型，完成96时段预测数据拉取；API响应超时或调用失败时5分钟内告警

**日前报价决策：**
- FR5: 系统通过多Agent协作流程，基于功率预测、电价预测和储能状态自动生成96时段报量报价建议及储能充放电计划
- FR6: 交易员可查看每个时段的AI报价建议及推荐理由（规则描述级可解释性）
- FR7: 交易员可对AI报价建议进行逐时段微调并记录调整理由
- FR8: 交易员完成审核后可确认最终报价方案，系统更新申报状态、记录确认信息、触发储能调度指令生成
- FR9: 系统识别负电价风险时段并预警；对配有储能的电站生成储能充电建议
- FR10: 系统记录交易员对AI建议的每次覆盖操作
- FR50: 交易员可在电站列表视图中批量查看所负责电站关键状态，支持筛选
- FR51: 系统展示每日申报状态跟踪，交易员可按状态筛选电站

**历史回测与收益分析：**
- FR11: 交易主管可选择指定电站和时间范围运行AI策略回测
- FR12: 系统计算AI策略（含储能联合优化）与实际执行策略的收益差异
- FR13: 系统对收益差异进行归因分析，对配有储能的电站单独展示储能贡献
- FR14: 交易主管可查看负电价专题分析
- FR15: 交易主管可生成并导出可视化回测报告（PDF格式）
- FR16: 高管可查看管理层回测报告

**收益看板与绩效分析：**
- FR17: 交易主管可按电站维度查看交易收益汇总（含储能套利收益）
- FR18: 交易主管可查看AI报价策略收益与实际执行收益的对比看板
- FR19: 交易主管可查看偏差考核费用跟踪数据
- FR20: 交易主管可按交易员维度查看绩效分析（含AI建议采纳率）
- FR21: 高管可查看跨所有电站的整体收益看板（只读）
- FR22: 交易员可查看自己负责电站的日/月交易收益及人工干预收益归因明细
- FR53: 交易主管可按是否配备储能对电站收益进行分组对比分析

**数据管理与导入：**
- FR23: 系统管理员可通过引导式向导配置电站基本参数及储能设备参数
- FR24: 系统管理员可配置省份市场规则
- FR25: 系统管理员可批量导入历史交易数据（Excel/CSV格式）
- FR26: 系统对导入数据进行质量校验并标记异常
- FR27: 系统管理员可查看异常数据列表，支持编辑修正、批量删除、标记已确认正常
- FR28: 系统通过省级电力交易中心公开API获取日前出清价格数据
- FR29: 系统接入电站实际出力数据（MVP文件导入为主）

**用户与权限管理：**
- FR30: 系统管理员可创建和管理用户账户
- FR31: 系统管理员可为用户分配角色（管理员/交易员/储能运维员/交易主管/高管只读）
- FR32: 系统管理员可配置交易员与电站的绑定关系
- FR33: 系统按角色和电站绑定关系控制数据访问范围
- FR52: 系统管理员可配置储能运维员与储能设备的绑定关系

**储能调度管理：**
- FR38: 系统自动生成日前96时段储能充放电计划
- FR39: 充放电计划生成时自动校验设备约束，超限指令自动拦截并告警
- FR40: 交易员可查看储能充放电计划及每个时段的策略建议和推荐理由
- FR41: 交易员可对储能充放电计划进行逐时段微调并记录调整理由
- FR42: 交易员确认储能调度计划后，系统生成调度指令发送给储能运维员
- FR43: 储能运维员可查看待执行调度指令，确认执行或拒绝并反馈异常
- FR44: 系统展示储能SOC状态
- FR45: 系统实现负电价时段储能充电策略
- FR46: 系统自动计算96时段最优充放电组合方案

**储能数据管理：**
- FR47: 系统管理员可通过文件导入储能历史运行数据
- FR48: 系统管理员可通过文件导入更新储能当前SOC状态
- FR49: 系统对导入的储能数据进行质量校验并标记异常

**多智能体协作：**
- FR54: 系统通过协调Agent统一管理各Agent的协作执行顺序和数据流转
- FR55: 交易员可查看Agent决策链路（输入摘要、输出结果、推理耗时、置信度评分）
- FR56: 系统通过中断-恢复机制暂停等待交易员审核
- FR57: 系统记录每次多Agent协作的完整执行链路作为审计日志
- FR58: Agent协作失败时自动降级为规则引擎生成报价建议
- FR59: 系统将多Agent协作流程状态持久化存储，支持从检查点恢复
- FR60: 系统提供Agent可观测性看板

**审计与合规：**
- FR34: 系统完整记录每笔AI报价建议的Agent级别决策链路
- FR35: 系统记录关键操作的审计日志
- FR36: 交易主管可按时间/电站/交易员维度检索审计日志
- FR37: 系统长期保存审计日志（≥3年），不可篡改

**总计功能性需求：53项**（FR1-FR60，跳过FR33后编号至FR60）

### 非功能性需求（Non-Functional Requirements）

**性能：**
- NFR1: 96时段AI报价建议生成延迟 < 30秒（含完整多Agent协作链路，P95基准）
- NFR2: 功率预测曲线加载时间 < 5秒（P95）
- NFR3: 历史回测（1年数据量）执行时间 < 10分钟
- NFR4: 收益看板页面加载时间 < 3秒（P95）
- NFR5: 系统支持同一实例内至少10个并发用户操作
- NFR20: 储能96时段充放电计划生成延迟 < 30秒（P95）
- NFR21: 储能调度指令从确认到运维员端展示延迟 < 10秒（P95）

**安全：**
- NFR6: 所有数据传输使用TLS 1.2+加密
- NFR7: 敏感数据采用AES-256加密存储，数据库字段级加密
- NFR8: 强密码策略+会话超时30分钟+连续5次登录失败锁定15分钟
- NFR9: 数据存储和处理完全在中国境内
- NFR10: 审计日志追加写入模式，不可修改或删除

**可扩展性：**
- NFR11: 单实例支持管理至少20个电站
- NFR12: 系统架构支持横向扩展（Phase 1 ≥3实例，Phase 2 ≥10实例）
- NFR13: 数据存储支持至少3年历史数据累积，查询劣化不超过50%
- NFR22: 储能调度引擎支持单实例内至少10台储能设备

**可靠性：**
- NFR14: 交易日系统可用性 ≥ 99.5%
- NFR15: 数据导入失败支持断点续传，续传成功率 ≥ 99%
- NFR16: AI引擎异常时60秒内切换至降级模式
- NFR23: 储能调度引擎异常时自动回退到无储能模式（成功率 ≥ 99%，通知延迟 ≤ 60秒）
- NFR24: 储能充放电指令设备约束硬校验，约束违反率 0%
- NFR26: RTO ≤ 4小时，RPO ≤ 1小时
- NFR27: 公开市场数据每日至少1次更新，数据陈旧超过24小时提示

**集成：**
- NFR17: 功率预测模型API接口标准化，新模型接入 ≤ 1人天
- NFR18: 外部数据源异常时30秒内切换至缓存数据
- NFR19: 数据导入支持Excel和CSV格式，单次 ≥ 100万条
- NFR25: 储能EMS数据导入接口标准化（MVP支持3家主流厂商数据格式转换）

**多Agent：**
- NFR28: 多Agent协作成功率 ≥ 95%
- NFR29: 单次多Agent协作总LLM Token消耗上限 20,000 tokens
- NFR30: Agent可观测性平台记录100%调用链路，数据保留 ≥ 90天
- NFR31: LLM模型热切换（仅需修改配置，切换操作 ≤ 10分钟）
- NFR32: Agent输入输出安全过滤，Prompt注入拦截率 ≥ 99%
- NFR33: Agent输出质量基线监控（月度评估，偏离 ≥ 20%触发告警）

**总计非功能性需求：33项**
